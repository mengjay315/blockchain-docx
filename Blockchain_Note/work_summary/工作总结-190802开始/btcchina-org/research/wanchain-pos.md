#### 1. 总发行量
_**`total = a + a*b + a*b^2 + ... = a / (1-b) (0 < b < 1)`**_  
- a: 第一个周期奖励量, 此处的周期是减产周期
- b: 每个周期的减产率
- epoch: 纪元, 可以当做一个难度调整周期, 每个epoch开始会选举RNP/EL/SL
- slot: 一个区块周期, 可以当做区块间隔, 由于在epoch开始一次性选举所有SL速度较慢, 当前SL的选举实际是在每个slot开始时选择的, 只需要选择一个  

#### 2. 奖励发放时机
每个epoch的第2K+1个slot, 发放前一个epoch的奖励, 分发的时候向下取整, 丢掉的部分计入下一周期奖励

#### 3. 第n个周期的总奖励
_**`P = a * b^n + S`**_
- S: 上一个周期的剩余奖励

#### 4. 一个epoch的总奖励
_**`G = P/N + T`**_
- N: 一个周期的epoch总数
- T: 一个epoch收集的总交易费

#### 5. 奖励来源
- 基础奖励: 基金会分配
- 剩余奖励: 上一个周期
- 交易费: epoch内的所有交易费

#### 6. 奖励分配方式
首先由奖励池收集总奖励, 然后按比例分发给三种角色, 再由每种角色扣除委托者的委托费后按比例分发到自己和委托者.

#### 7. 三种角色
- RNP: Random Number propose, 25个, 全部对外开放, 负责执行随机数生成协议
- EL: Epoch Leader, 50个, 基金会控制26个(>51%, 无奖励), 24个对外开放, 负责实现 Slot Leader 选取协议
- SL: Slot Leader, 根据RNP产生的随机数从EL中随机选择, 负责打包区块

#### 8. 三种角色的奖励比例
- RNP: 25/49
- EL: 12/49
- SL: 12/49

#### 9. wanchain参数
- 减产周期: 1年
- 减产率: 0.88
- 一个epoch的slot数: 12 * 1440
- 一个slot的周期: 5秒
- 一个epoch的周期: 一个epoch的slot数 * slot周期, 1天
- 一个减产周期的epoch数: 减产周期 / 一个epoch的周期, 365
- 第一个周期的奖励: 250万
- 总发行量: 第一个周期的奖励 / (1 - 减产率), 大约为2100万
- epoch的基础奖励: 第一个周期的奖励 / 一个减产周期的epoch数, 6849
- epochIDOffset: 当前epochID - 第一个epochID
- 减产次数: epochIDOffset / 一个周期的epoch数
- 第N个epoch的基础奖励: epoch的基础奖励 * 减产率^减产次数 + 上个周期的剩余奖励

#### 10. zchainx参数
- 减产周期: 10天
- 减产率: 0.9
- 总发行量: 10亿
- 预挖: 5亿
- 一个减产周期的epoch数: 10
- 第一个周期的奖励: (总发行量 - 预挖) * (1 - 减产率), 5000万
- epoch的基础奖励: 第一个周期的奖励 / 一个减产周期的epoch数, 500万
