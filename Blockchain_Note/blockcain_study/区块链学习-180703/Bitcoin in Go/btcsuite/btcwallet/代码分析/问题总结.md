# 问题总结
## 1. 传统的支付系统和比特币的支付系统账号的区别
    传统的中心化的支付系统，用户的账号是集中管理的，是存在一个中心化的服务器上，
    而区块链的去中心化的支付系统，账号是存在与本地的，一个账号下可以有好多地址，网络上传输的是地址，而不是账号，
    区块链钱包下可以有多个账号，一个账号又可以对应多个地址，
    钱包是用来管理账号的，账号是保存在本地的，而地址是管理交易的，

    这中账号和地址分开的设计理念，在一定成度上可以起到保护用户安全的作用，如果你频繁的使用某个账号转账，那么如果被某些人发现，可能会觉得你很有钱，可能会攻击你，
    账号和地址分开，交易使用的是地址，地址可以有很多，及时发生大额的转账，也不能确定是哪个账户，有一定的安全性，
    这个设计理念很好，但凡是都有两面性，这也导致了比特币常被用来洗黑钱，因为交易的是地址，无法追踪到是哪个账户进行了洗钱操作。

## 比特币的UTXO从这里开始：
    UTXO(unspand transaction output, 未花费的交易输出），也就是说比特币的账户只显示未花费的交易输出，
    什么意思呢？上面的默认账户原来有100个BTC,我给account1转了50个BTC,按道理转账后查询余额，default账户应该减少50(剩余的是100-50-交易费），account1应该是50BTC,而实际却不是
    传统的支付系统的特点是：从A->B 50， 此时余额应该是 A-50, B+50,是实时到账的
        而比特币的支付设计是UTXO,从default账户转出50给account1，是把default账户的100全拿走了，这100BTC其中50BTC是给account1的，另外50存在一个临时的地址里，该地址成为找零地址，
        此时再挖矿，再挖出一个block，就可以记录该笔交易，那么再查询账户余额，account1拿到了转账的50BTC,默认账户变成了99.99999627，这时挖出一个block对交易进行确认后，会获得50BTC的奖励，找零地址里的50BTC扣除交易费后又回到默认账户，所以会是99.99999627
